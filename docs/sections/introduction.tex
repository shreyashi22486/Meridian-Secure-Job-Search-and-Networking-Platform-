\section{Introduction}

The modern job market relies heavily on digital platforms where sensitive personal data — resumes, credentials, professional histories — flows between job seekers, recruiters, and employers. Despite handling such critical information, many platforms treat security as an afterthought, leaving user data vulnerable to breaches, credential theft, and unauthorized access.

\textbf{Meridian} addresses this gap. Built for the CSE~345/545 \textit{Foundations of Computer Security} course at IIIT Delhi, Meridian is a secure job search and professional networking platform where security is not bolted on but woven into every layer — from how passwords are stored to how resumes are encrypted on disk.

\subsection{Design Philosophy}

Our approach follows three guiding principles:

\begin{enumerate}[label=\textbf{\arabic*.}]
    \item \textbf{Defense in depth} — No single layer is trusted. Authentication checks are layered with session validation, device fingerprinting, and CSRF verification. File uploads pass through extension, MIME, and magic-byte checks before reaching encryption.
    \item \textbf{Secure by default} — Every endpoint requires authentication unless explicitly marked public. Passwords are hashed with Argon2id. Resumes are encrypted at rest with Fernet. Cookies are HttpOnly and Secure.
    \item \textbf{Transparency through auditing} — Every security-relevant action is logged with context (IP, user agent, timestamp), creating a forensic trail for incident response.
\end{enumerate}
