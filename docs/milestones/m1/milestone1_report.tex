\section{Overview}

This report covers Milestone~1 of the Meridian platform, due February~13, 2026. The milestone required three deliverables:

\begin{enumerate}
    \item Finalize the technology stack
    \item Configure HTTPS with TLS certificates
    \item Deploy a skeleton application on the VM
\end{enumerate}

All three goals are complete. This report documents the decisions made and the rationale behind each choice.

% ═══════════════════════════════════════════════════════════════════════
\section{Technology Stack}

Each technology was selected based on its security properties and development ergonomics.

\subsection{Backend — Python / FastAPI}

We chose FastAPI over Django and Express for three reasons:

\begin{itemize}
    \item \textbf{Dependency injection} — FastAPI's \texttt{Depends()} system cleanly separates authentication and RBAC from route logic. Every protected endpoint declares its security requirements as function parameters, making access control auditable at a glance.
    \item \textbf{Pydantic schemas} — All incoming data passes through typed schemas with validators. Malformed input is rejected before reaching business logic or the database.
    \item \textbf{Async-native} — Built on ASGI (Uvicorn), enabling non-blocking I/O for concurrent request handling without thread-safety concerns.
\end{itemize}

\subsection{Frontend — React 18 / Vite}

React was chosen for its component model and built-in XSS protection — all JSX expressions are automatically escaped. Vite provides fast hot module replacement during development and optimized production builds.

\subsection{Database — PostgreSQL 16}

PostgreSQL offers features critical to our security requirements:

\begin{itemize}
    \item \textbf{Native UUID columns} — Used as primary keys to prevent ID enumeration attacks
    \item \textbf{JSONB} — Flexible storage for audit log details without schema changes per event type
    \item \textbf{ACID compliance} — Transaction integrity for sensitive operations like account creation and session management
    \item \textbf{Parameterized queries} — Via SQLAlchemy ORM, structurally preventing SQL injection
\end{itemize}

\section{HTTPS Configuration}

All traffic is served over HTTPS with the following setup:

\subsection{Certificate Setup}

For the development environment on the course VM, we generated self-signed certificates using OpenSSL:

\begin{lstlisting}[language=bash]
openssl req -x509 -nodes -days 365 \
    -newkey rsa:2048 \
    -keyout ssl/private.key \
    -out ssl/certificate.crt \
    -subj "/CN=meridian.local"
\end{lstlisting}

\subsection{Nginx TLS Configuration}

Nginx is configured as a reverse proxy with TLS termination:

\begin{itemize}
    \item Listens on port 443 (HTTPS) with the self-signed certificate
    \item Port 80 (HTTP) redirects all traffic to HTTPS
    \item \texttt{Strict-Transport-Security} header enabled with 1-year max-age
    \item Proxies \texttt{/api/*} to the FastAPI backend (port 8000)
    \item Serves the React frontend build as static files
\end{itemize}

\subsection{Security Headers}

Even at this early stage, we configured security headers on all responses:

\begin{lstlisting}
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Strict-Transport-Security: max-age=31536000; includeSubDomains
Referrer-Policy: strict-origin-when-cross-origin
\end{lstlisting}

\section{Skeleton Application}

The skeleton deployment proves that all components work together end-to-end.

\subsection{What Was Deployed}

\begin{itemize}
    \item \textbf{FastAPI backend} with health check endpoint (\texttt{GET /api/health})
    \item \textbf{React frontend} with landing page, routing, and navigation shell
    \item \textbf{PostgreSQL database} with initial schema (users table)
    \item \textbf{Docker Compose} orchestrating all three services
    \item \textbf{Environment config} via \texttt{.env.example} documenting all required variables
\end{itemize}

\subsection{Verification}

The skeleton was verified by:
\begin{enumerate}
    \item Accessing the frontend via HTTPS (self-signed cert warning expected)
    \item Hitting the \texttt{/api/health} endpoint and confirming a 200 response
    \item Confirming PostgreSQL connectivity via Docker Compose logs
    \item Verifying all security headers are present on responses
\end{enumerate}
